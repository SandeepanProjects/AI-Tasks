The best way to understand this project is **not file-by-file first**, but **request-by-request**. Once you understand one user request flowing through the system, every file suddenly makes sense.

# Imagine User Asks

```text
Should a 55-year-old invest 80% of their portfolio in equities?
```

Let's trace this request through the entire codebase.

---

# STEP 1: FastAPI Receives Request

File:

```text
app/api/routes/chat.py
```

Example:

```python
@router.post("/chat")
async def chat(
    request: ChatRequest
):
    return await chat_service.chat(
        request.question
    )
```

Purpose:

```text
Receive Request
Validate Input
Call Business Logic
```

Think:

```text
Controller Layer
```

It should contain almost no business logic.

---

# STEP 2: JWT Authentication

File:

```text
app/auth/dependencies.py
```

Runs before chat endpoint.

Example:

```python
get_current_user()
```

JWT token arrives:

```text
Bearer eyJ...
```

Code verifies:

```python
JWTService.verify_token()
```

Output:

```python
{
    "user_id":"123",
    "role":"advisor"
}
```

Now system knows:

```text
Who is asking?
```

# STEP 8: Reranker

File:

```text
app/rag/reranker.py
```

Suppose we retrieved:

```text

```

Reranker scores:

```text

vs
Chunk
```

Example:

```text

Chunk 2 = 91%

Chunk 3 = 87%

Chunk 4 = 30%
```

Keeps:

```text
Top 5
```

This dramatically improves answer quality.

If you can explain that flow confidently and explain **why each component exists**, you'll already sound like a Senior AI Engineer rather than someone who just assembled libraries together.


This is exactly what interviewers care about.

Most candidates can say:

> "I used Redis, Qdrant, LangGraph, OpenAI."

Very few can explain:

> **"Why does this component exist? What problem does it solve? What would happen if we removed it?"**

That's what separates a Senior AI Engineer from someone who just followed tutorials.

# 1. FastAPI

### Why does it exist?

The AI system needs a way for external applications to communicate with it.

FastAPI exposes APIs.

```text
User
 ↓
POST /chat
 ↓
AI System
```

Without FastAPI:

```text
No way to interact with the system
```

---

### Interview Answer

> "FastAPI serves as the API gateway. It handles request validation, authentication, routing, and exposes the AI capabilities through REST endpoints."

---

# 2. JWT Authentication

### Why does it exist?

Financial data is sensitive.

We cannot allow anonymous users.

Need:

```text
Who is the user?
What role do they have?
```

Example:

```text
Advisor
Compliance Officer
Admin
```

JWT solves:

```text
Authentication
Authorization
```

---

### What happens without JWT?

Anyone can access:

```text
Portfolio Data
Investment Reports
Client Information
```

Security disaster.

---

### Interview Answer

> "JWT authentication ensures only authorized users can access financial recommendations and enables role-based access control."

---

# 3. PostgreSQL

### Why does it exist?

LLMs are not databases.

Need to store:

```text
Users
Chat History
Audit Logs
Configurations
```

Example:

```text
User: John

Question:
Should I retire at 60?

Answer:
...
```

Stored permanently.

---

### Why not Qdrant?

Qdrant stores vectors.

Not relational data.

---

### Interview Answer

> "PostgreSQL stores transactional and relational data such as users, conversations, audit logs, and application metadata."

---

# 4. Redis

### Why does it exist?

OpenAI calls are expensive.

Suppose:

```text
100 users ask:

What is SIP?
```

Without Redis:

```text
100 OpenAI calls
```

With Redis:

```text
1 OpenAI call
99 cache hits
```

Huge savings.

---

### Benefits

```text
Lower Cost
Lower Latency
Higher Throughput
```

---

### Interview Answer

> "Redis acts as a high-speed cache layer that reduces LLM costs and improves response latency by storing frequently accessed responses."

---

# 5. Embeddings

### Why do embeddings exist?

LLMs understand meaning.

Databases understand numbers.

Need a bridge.

Convert:

```text
Retirement Planning
```

into:

```text
[0.43, 0.91, 0.12 ...]
```

---

### Why?

Allows semantic search.

Example:

```text
Retirement Planning
```

can match:

```text
Post-retirement investment strategy
```

Even if words differ.

---

### Interview Answer

> "Embeddings transform text into vector representations, enabling semantic similarity search."

---

# 6. Qdrant

### Why does it exist?

Need to store millions of embeddings.

Traditional SQL databases are poor at vector search.

Qdrant is optimized for:

```text
Nearest Neighbor Search
```

---

### Example

Query:

```text
How should I invest near retirement?
```

Qdrant finds:

```text
Retirement Guide
Asset Allocation Study
```

based on meaning.

---

### Interview Answer

> "Qdrant acts as the vector database, enabling low-latency semantic retrieval of financial knowledge."

---

# 7. BM25

### Why does it exist?

Embeddings are not perfect.

Example:

```text
Fund XYZ 2025
```

Needs exact matching.

BM25 excels at:

```text
Keywords
Fund Names
Ticker Symbols
Numbers
```

---

### Why not use only vectors?

Vectors may miss:

```text
HDFC Fund
SBI Fund
```

exact identifiers.

---

### Interview Answer

> "BM25 complements vector search by improving retrieval for exact keywords and structured financial terminology."

---

# 8. Vector Search

### Why does it exist?

BM25 only understands words.

Vector search understands intent.

Example:

```text
How do I prepare financially for retirement?
```

Can retrieve:

```text
Retirement Savings Strategy
```

even without keyword overlap.

---

### Interview Answer

> "Vector retrieval captures semantic meaning and improves recall for conceptually similar content."

---

# 9. Reciprocal Rank Fusion (RRF)

### Why does it exist?

BM25 and Vector Search each make mistakes.

Combine both.

```text
BM25 Results
+
Vector Results
=
Better Results
```

---

### Interview Answer

> "RRF combines lexical and semantic retrieval results to improve overall retrieval quality."

---

# 10. Reranker

Most important component.

---

### Problem

Initial retrieval returns:

```text
30 documents
```

Many are mediocre.

Need:

```text
Best 5
```

---

### What does reranker do?

Scores:

```text
Question
vs
Document
```

More accurately than vector similarity.

---

### Effect

Often improves answer quality dramatically.

---

### Interview Answer

> "Cross-encoder reranking improves context precision by selecting the most relevant documents before generation."

---

# Staff-Level One-Line Summary

If an interviewer asks:

### "Why did you choose all these components?"

You can answer:

> "I designed the system as a layered architecture where FastAPI provides the API layer, PostgreSQL stores transactional data, Redis optimizes latency through caching, Qdrant enables semantic retrieval, hybrid search improves context quality, LangGraph orchestrates specialized financial agents, OpenAI performs reasoning and generation, and MLflow plus OpenTelemetry provide the observability and governance required for production AI systems."

That's the kind of explanation that demonstrates architectural thinking rather than just knowledge of tools.
